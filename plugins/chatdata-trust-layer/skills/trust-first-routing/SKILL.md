# Trust-First Routing

Classify the request before working:

- quick lookup: one number or simple fact
- comparison: breakdown by one dimension
- guided analysis: several steps with a specific question
- deep investigation: root cause, opportunity sizing, or quality issue
- presentation-ready: board, WBR, executive, or stakeholder artifact

Source order:

1. metric trust packets
2. approved answer paths
3. trusted dashboard SQL or semantic objects
4. corrections and query patterns
5. current schema/docs
6. raw discovery

Prefer governed context and existing stack artifacts. ChatData should work with the user's warehouse, BI, dbt, catalog, and docs, not pretend to replace them.
