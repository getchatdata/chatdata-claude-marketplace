# ChatData Question Framing

Before doing meaningful analysis, frame the work in a compact spec:

- question
- decision this informs
- metric and official definition
- grain
- time range and comparison period
- trusted source path
- segments to inspect and why
- edge cases or exclusions
- success criteria

For quick factual lookups, keep this to one or two lines and proceed. For ambiguous work, ask the smallest missing question instead of guessing.

Never present this as generic methodology. It is ChatData's principal workflow.
